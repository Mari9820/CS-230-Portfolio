# CS-230 Portfolio

## Portfolio Reflection

The client for this project was The Gaming Room. They wanted a web-based version of their game, Draw It or Lose It, that could support multiple teams and players while preventing duplicate game instances. The software also needed to be scalable, secure, and work across different operating systems. My job was to create a software design document that recommended the best platform and architecture before development began.

I think one thing I did well was comparing different operating systems and explaining why Linux running in a cloud environment would be the best option for this project. Instead of just choosing one, I looked at the strengths and weaknesses of each platform and supported my recommendation with reasons related to cost, security, and scalability.

Working through the design document before writing code was really helpful because it made me think through the project step by step. It forced me to understand the client's requirements first instead of jumping straight into programming. Having a clear plan made it easier to understand how the different parts of the application would work together.

If I could go back and improve one part of my work, I would add more detail about security and cloud deployment. Throughout this course I learned more about cloud platforms and how important scalability and security are, so I think I could explain those design decisions more clearly now.

When designing the software, I focused on the client's needs first. I made sure my recommendations matched their goals of creating a reliable, scalable, and easy-to-maintain application. I learned that understanding what the client actually needs is just as important as writing good code because even well-written software won't be successful if it doesn't solve the right problem.

As I continue developing software, I plan to spend more time planning before coding. Creating diagrams, documenting requirements, and evaluating different technologies helped me understand the project much better. In future projects, I will continue using those strategies because they make the development process more organized and reduce problems later on.
